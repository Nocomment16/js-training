Exercises

Closure (1/3)

  1. create main.js file in the main directory
  2. create personFactory function in the main.js file
  3. create details object inside the personFactory function with properties: firstName, lastName, accounts list
  4. firstName, last name should have some string values assigned
  5. accounts list should have single account object inside with some example balance (number) and currency (string)

Closure (2/3)

  1. personFactory should return object with properties firstName, lastName and method sayHello
  2. firstName and lastName should take its values from inner details object
  3. sayHello method should return a string with firstName, lastName and number of accounts assigned to an object created by personFactory

Closure (3/3)

  1. create an object using personFactory function and assign it to a variable
  2. call sayHello method inside of a console.log method in order to print the result of the method
  3. attach the main.js script to the index.html file
  4. open index.html in the browser and check the results in the browser console
  5. run the script in the node.js with the following commad: node main.js - check the results

Module Pattern (1/2)

  1. rename personFactory to person
  2. refactor code in order to use module pattern with IIFE
  3. move accounts from details object to object returned by IIFE
  4. add local calculateBalance() method
  5. add addAccount(account) method to module object
  6. update sayHello method in order to print person's total balance additionaly

Module Pattern (2/2)

  1. call sayHello() on person module
  2. call addAccount(someAccount) on person module
  3. call sayHello() on person once again to see if balance is updated
  4. check the result in the browser and in node.js

Constructor Function (1/1)

  1. create Account constructor function in the main.js file
  2. constructor function should take balance and currency as parameters
  3. replace one of the account creation (literal object expression) with constructor function call
  4. check the result in the browser and in node.js

ES6 Syntax (1/2)

  1. replace Account constructor function with the Account class
  2. constructor of the Account class should take balance and currency as parameters
  3. replace person module with Person class
  4. constructor of the Person class should take firstName, lastNames and accounts list as parameters
  5. replace local calculateBalance() method with a Person class method _calculateBalance()
  6. in the _calculateBalance() use of operator to iterate over array

ES6 Syntax (2/2)

  1. replace the addAccount method with the Person class method addAccount
  2. replace the sayHello method with the Person class method sayHello, use string interpolation inside
  3. implement Person class method filterPositiveAccounts() which finds all accounts with a positive balance (use filter with combination of arrow function)
  4. define new person with const keyword, do some logging before / after adding accounts, print the result of filterPositiveAccounts() method

Promises (1/2)

  1. add number property to Account class
  2. add findAccount(accountNumber) method to Person class (use Array.prototype.find)
  3. add withdraw(accountNumber, amount) method to Person class
  4. withdraw method returns a promise (create promise with new Promise(resolve, reject) and return it)
  5. promise is resolved after 3 seconds (use setTimeout(callback)) when the account is found (use findAccount method) and the given amount is higher than foundAccount.balance

Promises (2/2)

  1. when promise is resolved, the account balance is updated and success message appears (with account number, new balance, amount of withdrawn money)
  2. promise is rejected either when the account can't be found or when the requested amount is bigger than account balance
  3. in case of rejection, there is an appropriate message returned
  4. update all accounts creation (Account constructor calls) with account numbers
  5. withdraw some money for different cases (use person.withdraw(accNumber, amount).then(...).catch(...)
