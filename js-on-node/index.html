node.js - is a JavaScript runtime built on Chrome's V8 JavaScript engine. Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient.

Running js on node


node <path-to-file>

npm - node package manager makes it easy for JavaScript developers to share and reuse code, and it makes it easy to update the code that you're sharing.

Installation

npm is automatically installed with node.js

Check the installed node and npm versions:


node -v && npm -v

Install packages locally (into local node_modules dir):


npm install <package-name>
// shorthand
npm i <package-name>

Install packages globally:


npm install <package-name> -g
// check packages installed globally
npm list -g
// top level packages only
npm list -g --depth=0

Managing packages with package.json

The package.json file provides convinient way to manage local dependencies

In order to create package.json file for a project:


npm init

Install all dependencies from package.json:


npm install // called in the directory with package.json file

Ecerise 6. - Manage dependencies with npm

yarn - js dependency manager with package caching and deterministic algorithm for installs

Installation

via installer

via npm (not recommended):


npm install -g yarn

Install packages locally (into local node_modules dir):


// automatically adds to dependencies in package.json
yarn add install <package-name>

Install packages globally:


yarn add global <package-name>
// check packages installed globally
yarn global ls

Managing packages with package.json

yarn (like npm) can use package.json file to manage local dependencies

In order to create package.json file for a project:


yarn init

Install all dependencies from package.json:


yarn
// or
yarn install // called in the directory with package.json file
